## 12. [P2] Go ↔ Rust: Steam Networking Sockets transport bridge

> **Architecture note (multiplexed IPC).** Peer bytes flow over the existing
> single Rust↔Go IPC channel (the one defined in §8.3 / `ipc.rs` /
> `IPCBridge`), multiplexed by an opaque `peerId` token. The phrase "paired
> Unix-socket/named-pipe leg" in 12.1 refers to that one existing channel —
> NOT a fresh per-peer socket. Three new notification kinds carry traffic:
> `new_peer_transport` (Rust → Go, peer joined), `peer_message` (Rust → Go,
> inbound bytes), `peer_disconnected` (Rust → Go, peer left); two request
> methods carry outbound work: `send_peer_message` and `close_peer`. The
> existing `ipc.rs::push_notification` + `IPCBridge.OnEvent` machinery
> already supports this pattern. Rationale: simpler than a per-peer socket
> per friend, single channel to manage, and game traffic over an RTS at our

- Lobby chat / voice chat / in-game text chat.
- Per-session passcode authentication on Direct connect (a possible follow-up enhancement, but not in this change).
